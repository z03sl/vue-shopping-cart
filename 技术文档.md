# 基于 Vue 框架的购物车列表应用 — 技术文档

## 1. 项目介绍

本系统为单页购物车原型：展示模拟商品列表，支持加入购物车、数量增减与输入校验、按 ID 移除、单选/全选、按勾选项计算合计金额与件数，并通过 LocalStorage 在刷新后保留购物车数据。

## 2. 技术架构

### 2.1 技术选型

| 项目       | 说明                          |
| ---------- | ----------------------------- |
| 构建工具   | Vite                          |
| 框架       | Vue 3（`<script setup>`）     |
| 状态管理   | Pinia（对应 Vue 3 的集中状态） |
| 持久化     | `localStorage`                |
| 样式       | 基础 CSS（`scoped`）          |

### 2.2 目录结构

```
vue-shopping-cart/
├── index.html
├── src/
│   ├── main.js              # 入口，注册 Pinia
│   ├── App.vue              # 根组件：挂载时读 LocalStorage；深度 watch 写入
│   ├── style.css            # 全局样式
│   ├── data/goods.js        # 模拟商品 goodsList
│   ├── stores/cart.js       # Pinia：state / getters / actions
│   └── components/
│       ├── ProductList.vue  # 商品列表（v-for）
│       ├── Cart.vue         # 购物车容器、空状态 v-if、全选、合计
│       └── CartItem.vue     # 单项：复选框、数量、移除
```

### 2.3 组件关系与数据流

- **Props / 事件**：`Cart` 将 `item` 以 props 传给 `CartItem`；业务操作在子组件内通过 `useCartStore()` 调用 actions（与任务书中「子组件通过 store 与父级数据一致」等价于集中状态管理）。
- **向下**：商品数据来自 `data/goods.js`；购物车项结构在 store 中维护。
- **计算属性**：总金额、总件数、是否全选在 `stores/cart.js` 的 getters 中实现，便于缓存与模板绑定。

## 3. 核心实现说明

### 3.1 添加商品

在 `addToCart` 中若已存在同 `id` 则 `quantity + 1`（受 `stock` 限制）；否则 `push` 新项，`quantity` 初值为 1，并带 `checked: true` 与 `stock` 副本。

### 3.2 数量与移除

- `incrementQty` / `decrementQty`：在库存与「数量为 1 时禁用减号」规则下更新数量；`removeFromCart` 用 `findIndex` + `splice` 按 `id` 删除。

### 3.3 全选与合计

- 每项含 `checked`；全选使用计算属性 `get`/`set` 绑定复选框，调用 `setAllChecked`。
- `cartTotalPrice`、`cartTotalCount` 仅统计 `checked === true` 的项。

### 3.4 持久化

- `onMounted` 中调用 `loadFromStorage()` 读取 `localStorage`。
- `App.vue` 中对 `cart.cartItems` 使用 `watch(..., { deep: true })`，变化时写回 `localStorage`。

### 3.5 体验与校验

- 数量为 1 时「−」禁用；达到库存时「+」禁用。
- 数量输入框失焦时校验整数范围与库存（`setQuantity`）。
- 列表增删使用 `<transition-group>` 做简单过渡。

## 4. 运行与构建

```bash
cd vue-shopping-cart
npm install
npm run dev
```

浏览器访问终端提示的本地地址。生产构建：

```bash
npm run build
npm run preview
```
