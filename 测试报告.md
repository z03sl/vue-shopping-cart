# 购物车应用 — 测试报告

## 测试环境

- 浏览器：Chrome（建议同时用 Firefox 做兼容性抽查）
- 构建：`npm run dev` 本地开发服务器

## 测试用例与结果

| 编号 | 用例描述                         | 预期结果                         | 结果   |
| ---- | -------------------------------- | -------------------------------- | ------ |
| T1   | 商品列表展示                     | 显示名称、价格、图片、库存       | 通过   |
| T2   | 首次加入购物车                   | 购物车出现该项，数量为 1         | 通过   |
| T3   | 再次加入同一商品                 | 数量 +1，不超过库存              | 通过   |
| T4   | 加号至库存上限                   | 提示「已达到库存上限」或按钮禁用 | 通过   |
| T5   | 减号（数量>1）                   | 数量 −1                          | 通过   |
| T6   | 数量为 1 时点减号              | 按钮禁用，需用「移除」删除       | 通过   |
| T7   | 移除                             | 该项从列表消失                   | 通过   |
| T8   | 购物车为空                       | 显示空状态提示                   | 通过   |
| T9   | 单选/取消                        | 合计金额、件数随勾选变化         | 通过   |
| T10  | 全选/取消全选                    | 所有项勾选状态一致               | 通过   |
| T11  | 刷新页面                         | 购物车数据与刷新前一致         | 通过   |
| T12  | 数量输入非法（空、小数、超库存） | 提示并保持合法数量             | 通过   |
| T13  | 购物车为空时仅显示空提示         | 不显示全选工具条                 | 通过   |

## 典型问题与处理

1. **刷新丢数据**：在根组件对购物车数组做深度 `watch` 写入 `localStorage`，并在 `mounted` 中读取恢复。  
2. **合计与勾选不一致**：合计仅统计 `checked === true` 的项，与全选、单选逻辑统一在 Pinia getters 中维护。
